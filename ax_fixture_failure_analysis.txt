====================================================================================================
KEY TAKEAWAYS
====================================================================================================

1. SIGNAL RATIO CRISIS:
   - ChatGPT: 13% signal (most is sidebar navigation)
   - Discord: <5% signal (server lists, channels, members dominate)
   - No conversation content visible in empty/new chat states

2. TOKEN BUDGET EXPLOSION:
   - Claude-4: 1MB JSON ≈ 250K tokens just for structure
   - Gemini browser: 1.5MB with Chrome scaffold
   - Would exceed most LLM context windows before adding actual prompt

3. MISSING SEMANTIC STRUCTURE:
   - Can't distinguish user messages from assistant messages
   - Timestamps buried or missing
   - No clear conversation turn boundaries
   - Speaker attribution unclear

4. WRONG INFORMATION SURFACE:
   - LLM sees: navigation buttons, settings, sidebar lists
   - LLM doesn't see: actual message content, conversation context
   - Historical UI chrome outweighs current input focus

5. DEPTH COMPLEXITY:
   - Deep nesting (15-20 levels) obscures hierarchical relationships
   - Structural wrappers (AXGroup, AXHostingView) add no value
   - Adjacent text fragments need manual concatenation

CONCLUSION:
Feeding raw AX trees to an LLM is like asking it to read a website by giving
it the entire browser DevTools DOM inspector output. The signal is there, but
buried under 10-100x noise. Purpose-built extractors are essential.
